#  2. Linux Basics Notes

## Why Learn Linux for DevOps?
- 90%+ of cloud servers, containers (Docker), orchestration (Kubernetes) run on Linux.
- Commands are essential for scripting, automation, troubleshooting.

## Linux File System Structure (Hierarchy)
Everything starts from root `/`. It's a tree-like structure (invert of Windows C:\).

Key Directories:
- `/` : Root directory (top-level).
- `/bin` : Essential binaries/commands (ls, cd, etc.).
- `/etc` : Configuration files (system settings).
- `/home` : User home directories (e.g., /home/pavan).
- `/var` : Variable data (logs, temporary files – /var/log).
- `/usr` : User programs and data (installed software).
- `/tmp` : Temporary files (cleared on reboot).
- `/opt` : Optional/add-on software.
- `/dev` : Device files (hard drives, etc.).
- `/proc` : Virtual filesystem for process info.
- `/boot` : Bootloader files (kernel, etc. – don't touch!).

Command to view structure: `tree -L 2 /` (if tree installed) or `ls -l /`.

## Basic Linux Commands Cheat Sheet
| Command     | Description                          | Example                  | Options/Notes |
|-------------|--------------------------------------|--------------------------|---------------|
| pwd        | Print Working Directory             | pwd                     | Shows current path |
| ls         | List files/directories              | ls -l (detailed), ls -a (hidden) | ls -la (both) |
| cd         | Change Directory                    | cd /home, cd .. (up), cd ~ (home) | cd (go home) |
| mkdir      | Make Directory                      | mkdir newfolder         | mkdir -p /path/to/nested |
| rmdir      | Remove empty Directory              | rmdir folder            | Use rm -r for non-empty |
| touch      | Create empty file / update timestamp| touch file.txt          |               |
| cp         | Copy files/directories              | cp file.txt newfile.txt | cp -r folder newfolder |
| mv         | Move/Rename                         | mv file.txt /path/      |               |
| rm         | Remove files/directories            | rm file.txt, rm -r folder | -f to force |
| cat        | View file content                   | cat file.txt            |               |
| echo       | Print text                          | echo "Hello" > file.txt |               |
| man        | Manual for commands                 | man ls                  |               |

# 3. Linux Commands
* `pwd`                        # Print current working directory
* `ls -la`                     # List all files (including hidden) with details
* `cd /`                       # Change directory to root
* `ls`                         # List contents of current directory
* `cd ~`                       # Change directory to home
* `mkdir devops-practice`      # Create directory named devops-practice
* `cd devops-practice`         # Change directory to devops-practice
* `touch testfile.txt`         # Create an empty file named testfile.txt
* `echo "Hello DevOps!" > testfile.txt`   # Write "Hello DevOps!" to testfile.txt (overwrite if exists)
* `cat testfile.txt`           # Display contents of testfile.txt
* `cp testfile.txt backup.txt` # Copy testfile.txt to backup.txt
* `mv backup.txt renamed.txt`  # Rename backup.txt to renamed.txt
* `ls -l`                      # List files in long format
* `rm renamed.txt`             # Delete the file renamed.txt
* `cd ..`                      # Change to parent directory
* `rmdir devops-practice`      # Remove devops-practice directory (only if empty)
* `rm -r devops-practice`      # Remove directory and its contents recursively (use with caution)
